1.2 Cara Kerja Aplikasinya

Secara garis besar, aplikasi ini jalan seperti ini:

  1. User buka browser → akses localhost:8000
  2. Router Laravel ngecek URL yang diminta, terus tentuin controller mana yang harus jalan
  3. Middleware ngecek dulu: user udah login belum? Role-nya apa? Boleh akses halaman ini nggak?
  4. Controller ambil data dari database (pakai query SQL), olah datanya, terus kirim ke view (Blade)
  5. Blade render HTML yang akhirnya ditampilkan ke browser user

1.3 Kenapa Pakai Raw SQL?

Sebagian besar query di project ini ditulis langsung sebagai SQL mentah, bukan pakai Eloquent ORM bawaan Laravel. Contohnya:

// Ini yang dipake di project — raw SQL langsung
$users = DB::select("SELECT * FROM users WHERE role = ? ORDER BY created_at DESC", ['user']);

// Kalau pakai Eloquent ORM, jadinya gini:
$users = User::where('role', 'user')->orderBy('created_at', 'desc')->get();

Alasannya: dengan raw SQL, kita bisa lihat persis query apa yang dijalankan, gampang bikin JOIN yang kompleks, dan performanya lebih predictable.

Routing itu kayak "peta" aplikasi — ngatur URL mana yang harus dilayani oleh controller apa. Middleware itu kayak "satpam" — ngecek dulu boleh nggak sih request ini masuk.

3.1 Middleware yang Ada

3.1.1 AuthCheck — "Udah login belum?"

// File: app/Http/Middleware/AuthCheck.php

public function handle(Request $request, Closure $next)
{
    // Cek apakah ada session user_id
    if (!session('user_id')) {
        // Kalau minta JSON (AJAX), balikin 401
        if ($request->expectsJson()) {
            return response()->json(['error' => 'Unauthenticated.'], 401);
        }
        // Kalau request biasa, lempar ke halaman login
        return redirect()->route('login');
    }

    // Kalau ada session-nya, lanjut ke controller
    return $next($request);
}

Simpel: kalau nggak ada user_id di session, tendang ke halaman login.

3.1.2 AdminOnly — "Khusus admin aja"

public function handle(Request $request, Closure $next)
{
    // Harus login DAN role harus 'admin'
    if (!session('user_id') || session('role') !== 'admin') {

}

4.1 Proses Login

4.1.1 Validasi Input

Pertama, cek dulu username sama password udah diisi belum:

public function login(Request $request)
{
    $request->validate([
        'username' => 'required',
        'password' => 'required',
    ], [
        'username.required' => 'Email dan password tidak boleh kosong.',
        'password.required' => 'Email dan password tidak boleh kosong.',
    ]);

4.1.2 Cocokkan Username/Email

User bisa login pakai email ATAU username. Makanya query-nya cek dua kolom:

    $identifier = trim($request->username);

    // Cari user yang email-nya ATAU username-nya cocok
    $user = User::where('email', $identifier)
                ->orWhere('username', $identifier)
                ->first();

    // Cek apakah user ketemu DAN password-nya bener
    if (!$user || !Hash::check($request->password, $user->password)) {
        return back()->withErrors(['login' => 'Email atau password salah.']);
    }

Hash::check() itu fungsinya membandingkan password yang diketik user sama password yang udah di-hash (diacak) di database. Kita nggak pernah nyimpan password asli.

4.4 Reset Password

public function resetPassword(Request $request)
{
    $email    = trim($request->input('email', ''));
    $password = $request->input('password', '');
    $confirm  = $request->input('confirm_password', '');

    // Validasi
    if (strlen($password) < 8) {
        return response()->json(['status' => 'error',
            'message' => 'Password minimal 8 karakter.']);
    }
    if ($password !== $confirm) {
        return response()->json(['status' => 'error',
            'message' => 'Konfirmasi password tidak cocok.']);
    }

    // Cari user, update password
    $user = User::where('email', $email)->first();
    if (!$user) {
        return response()->json(['status' => 'error',
            'message' => 'Email tidak terdaftar.']);
    }

    $user->password = Hash::make($password);
    $user->save();

    return response()->json(['status' => 'success',
        'message' => 'Password berhasil diubah. Silakan login.']);
}

            DB::delete('DELETE FROM lamaran WHERE user_id = ?', [$user_id]);

private function doApiCall(string $apiKey, string $model, array $messages): ?string
{
    $payload = [
        'model'       => $model,
        'messages'    => $messages,
        'temperature' => 0.2,  // rendah = jawaban lebih konsisten
        'max_tokens'  => 2000,
    ];

    // Kirim request ke OpenRouter pakai cURL
    $ch = curl_init(config('openrouter.base_url'));
    curl_setopt_array($ch, [
        CURLOPT_POST           => true,
        CURLOPT_POSTFIELDS     => json_encode($payload),
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_TIMEOUT        => 60,
        CURLOPT_HTTPHEADER     => [
            'Content-Type: application/json',
            'Authorization: Bearer ' . $apiKey,
            'HTTP-Referer: http://localhost:8000',
            'X-Title: DoleKerjo AI Matching',
        ],
    ]);

    $response = curl_exec($ch);
    $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    curl_close($ch);

    if ($httpCode === 429) return null; // rate limit → coba model lain
    if ($httpCode !== 200) return null; // error lain

    $data = json_decode($response, true);
    return $data['choices'][0]['message']['content'] ?? null;
}

8.3.6 Parse Jawaban AI

private function parseAIResponse(string $raw, array $jobs): array
{
    // Bersihin kalau AI-nya ngirim markdown code block
    $clean = preg_replace('/```(?:json)?\s*/i', '', $raw);

    // Coba parse JSON-nya
    $parsed = json_decode($clean, true);
    if (!is_array($parsed)) {
        // Kalau gagal, coba ekstrak pakai regex
        if (preg_match('/\[.*\]/s', $clean, $matches)) {
            $parsed = json_decode($matches[0], true);
        }
    }
    if (!is_array($parsed)) return [];

    // Cocokkan job_id dari AI dengan data job yang sebenarnya
    $jobMap = [];
    foreach ($jobs as $j) { $jobMap[$j->id] = $j; }

    $results = [];
    foreach ($parsed as $item) {
        $jobId = $item['job_id'] ?? null;
        if (!$jobId || !isset($jobMap[$jobId])) continue;

        $j = $jobMap[$jobId];
        $results[] = [
            'id'             => $j->id,
            'nama_posisi'    => $j->nama_posisi,
            'match_percent'  => min(max(intval($item['score'] ?? 0), 0), 100),
            'reasoning'      => $item['reasoning'] ?? '',
            'matched_skills' => $item['matched_skills'] ?? [],
            'gaps'           => $item['gaps'] ?? [],
        ];
    }

    // Urutkan dan ambil top 10
    usort($results, fn($a, $b) => $b['match_percent'] <=> $a['match_percent']);
    return array_slice($results, 0, 10);
}

use Illuminate\Foundation\Auth\User as Authenticatable;

class User extends Authenticatable
{
    // Field yang boleh diisi massal (pakai create() atau fill())
    protected $fillable = [
        'nama', 'username', 'email', 'password',
        'telepon', 'lokasi', 'bio', 'foto_profil',
        'tanggal_lahir', 'gender', 'pendidikan',
        'jurusan', 'role',
    ];

    // Field yang disembunyikan kalau model di-serialize ke JSON
    protected $hidden = ['password'];
}

// Cara pakai (di AuthController):
$user = User::where('email', $identifier)->first();
// Ini sama aja dengan: DB::selectOne("SELECT * FROM users WHERE email = ?", [$identifier])

13.1 File config/tipe_map.php

File ini nyimpen "kamus" tipe lowongan. Dipakai di banyak controller biar label dan warna badge konsisten.

return [
    'full-time'   => ['class' => '',        'label' => 'Full Time'],
    'part-time'   => ['class' => 'parttime', 'label' => 'Part Time'],
    'remote'      => ['class' => 'remote',   'label' => 'Remote'],
    'hybrid'      => ['class' => 'hybrid',   'label' => 'Hybrid'],
    'contract'    => ['class' => 'contract', 'label' => 'Contract'],
    'partnership' => ['class' => 'partner',  'label' => 'Partnership'],
];

// Cara pakai di controller:
$tipeMap = config('tipe_map');
$map = $tipeMap[$job->tipe] ?? ['class' => '', 'label' => ucfirst($job->tipe)];
$job->tipe_label = $map['label'];   // "full-time" → "Full Time"
$job->tipe_class = $map['class'];   // "remote" → "remote" (CSS class)

13.3 File bootstrap/app.php

Tempat daftarin middleware alias biar bisa dipanggil pakai nama pendek di route:

return Application::configure(basePath: dirname(__DIR__))
    ->withRouting(
        web: __DIR__.'/../routes/web.php',
        commands: __DIR__.'/../routes/console.php',
        health: '/up',
    )
    ->withMiddleware(function (Middleware $middleware): void {
        // Daftarin nama alias buat middleware custom kita
        $middleware->alias([
            'auth.check'     => \App\Http\Middleware\AuthCheck::class,
            'admin.only'     => \App\Http\Middleware\AdminOnly::class,
            'recruiter.only' => \App\Http\Middleware\RecruiterOnly::class,
        ]);
    })
    ->withExceptions(function (Exceptions $exceptions): void {
        //
    })->create();

Setelah didaftarkan di sini, kita bisa pakai ->middleware('admin.only') di route, alih-alih nulis class path lengkap.
